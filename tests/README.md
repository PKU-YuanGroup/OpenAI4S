# Tests

[中文说明](README_zh.md)

The offline correctness gate for OpenAI4S. `uv run pytest` runs every module here against deterministic fakes, and it has to stay green: the provider-neutral agent engine, the Host services, the persistent Python/R kernel protocol, the repositories, the security boundaries, the tools, the Skills, and the Web composition. Reusable scenarios and scored evaluations are a separate layer and live in [`../harness/`](../harness/).

## Offline contract

- `uv run pytest` must not need a live LLM, an API key, network, a GPU, an SSH host, a Docker daemon, a browser, or a lab system. [`conftest.py`](conftest.py) points `~/.openai4s` at a temporary directory for every test and installs a fake provider and key.
- A test that needs an external resource has to opt in through a marker registered in `pyproject.toml`; [`test_marker_policy.py`](test_marker_policy.py) guards that rule.
- Captured inputs and byte-sensitive samples live under `fixtures/`; tests must not silently rewrite them.
- Network, subprocess, provider, clock, UUID, and filesystem boundaries are mocked or confined. The only exception is a smoke program that says so and is invoked separately.
- Run one module with `uv run pytest tests/test_kernel.py`, one case with `uv run pytest tests/test_agent.py::test_max_turns_stop`, or the full gate with `uv run pytest`.

## Support and smoke files

| File | Responsibility |
| --- | --- |
| [`conftest.py`](conftest.py) | Sets up import paths, gives every test its own data directory, configures a fake LLM provider and key, cleans up the `Store` afterwards, and holds the shared pytest fixtures. |
| [`browser_smoke.mjs`](browser_smoke.mjs) | Drives a real browser against the running Gateway UI and its streamed interaction paths. You invoke it separately from local pytest; the normal PR CI workflow runs it automatically. |

## Test modules

Every `test_*.py` file is listed below, with the reason the module exists rather than an inventory of its cases. Most modules also carry extra regression cases around failure, restart, redaction, and concurrency that the line does not name.

| File | Why you would open this file |
| --- | --- |
| [`test_action_ledger_repository.py`](test_action_ledger_repository.py) | Four assertions about what the Ledger storage refuses to do: overwrite a group or an event, split an atomic tool group, or rewrite a terminal execution attempt. The column migration is additive. |
| [`test_action_ledger_runtime.py`](test_action_ledger_runtime.py) | The other half of the Ledger: what the running loop actually writes into it. Arguments are redacted before they land, a branch inherits only its checkpointed parent prefix, an interrupted native group is closed atomically, and the history rebuilds after the store is reopened. |
| [`test_action_routing_eval.py`](test_action_routing_eval.py) | Less an assertion suite than a scorer. It replays offline routing fixtures through the Tool/Code/Finalize router and emits a failure and confusion report for a human to review. |
| [`test_action_timeline_service.py`](test_action_timeline_service.py) | Open this when the Timeline shows a user something it should not. The projection is public, so the tests are mostly about what it withholds — raw payloads, secrets, provider IDs — and about what it must not withhold, such as an earlier tool failure. |
| [`test_actions.py`](test_actions.py) | Reply parsing, shared by both outer loops. The precedence rules are pinned here: a native call beats a code fence, an unclosed fence is never executable, and only the first Cell in document order runs. |
| [`test_admet_genetic.py`](test_admet_genetic.py) | The bundled ADMET genetic Skill: that it is discoverable, that its helpers aggregate deterministically, and that the dashboard it emits escapes script and HTML delimiters. |
| [`test_agent.py`](test_agent.py) | The broadest module in the suite — the offline outer loop end to end. Code-as-Action cycles, an R Cell that soft-fails into an observation when R is absent, token estimation, compaction that keeps a Cell and its observation in one atomic segment, and the delegation caps. A change that breaks the loop usually breaks here first. |
| [`test_agent_control.py`](test_agent_control.py) | Native tool batches under stress. Every call in a batch reaches a result even when one invocation fails or the turn is cancelled mid-batch. Independent read-only calls may run in parallel, but a mutating call is a barrier for everything after it. |
| [`test_agent_engine.py`](test_agent_engine.py) | `AgentEngine` on its own, driven through fake ports. The last test is the point of the module: the engine must not import runtime infrastructure. The rest pin routing precedence, replay-safe history grouping, and the exact moments at which cancellation wins. |
| [`test_agent_hybrid.py`](test_agent_hybrid.py) | Two tests on the hybrid `Agent` facade: a native call beats code and its canonical history survives into the next turn, and a reused agent clears the previous submission before a new task. |
| [`test_agent_profile_repository.py`](test_agent_profile_repository.py) | Named agent profiles in SQLite, mostly the awkward parts — legacy serialization of falsy values, JSON decoding edges in listings, and the read-then-write gap an upsert has to survive. |
| [`test_agent_runtime.py`](test_agent_runtime.py) | The local adapters between the pure engine and real infrastructure. Two rules dominate: a native parse error or a limit must never dispatch, and it must never drop a tool result either. Compaction expands its tail to keep an assistant tool group atomic, and a circuit breaker stops repeated low-yield compaction. |
| [`test_analysis_skills.py`](test_analysis_skills.py) | The bundled analysis Skills are executed here, not just listed. Their data audit catches group leakage, the AUC is tie-aware, and the bootstrap is deterministic. |
| [`test_annotation_repository.py`](test_annotation_repository.py) | Image annotations. Three tests, and the one that matters is ordinal allocation under concurrent pins — it has to be atomic. |
| [`test_artifact_control_tools.py`](test_artifact_control_tools.py) | Artifact restore is the dangerous operation, so most of this module is about refusing it: a corrupt snapshot, an untrusted ID, a workspace that drifted since the version was cut. Restore stays behind approval and rolls back a failed store write. |
| [`test_retrieval_provenance.py`](test_retrieval_provenance.py) | A retrieved record says when it was fetched and hashes what upstream returned, so a rerun that quietly changed is distinguishable from one that did not — and that envelope reaches the artifact derived from it. |
| [`test_artifact_env_provenance.py`](test_artifact_env_provenance.py) | An artifact records the environment that produced it: an R cell's artifact stops carrying a Python package list, a selected conda env is named rather than the daemon's, and an unreadable interpreter reports absence instead of borrowing this process's packages. |
| [`test_response_schemas.py`](test_response_schemas.py) | The HTTP surface has a shape that can be checked. Response shapes are captured from real responses rather than written by hand, so the frozen file cannot describe a response the code does not produce; the tests pin the shape algebra (an optional field must not stay required, an observed null makes a field nullable rather than optional) and the distinction the gate exists to make — a removed guarantee is BREAKING, a new field is additive. |
| [`test_server_errors_module.py`](test_server_errors_module.py) | The HTTP failure type is importable without importing the gateway. A prerequisite for decomposing `Handler._api`, not tidying: `GatewayError` sat ~5,800 lines below gateway's own imports, so a route module importing it from there failed the daemon at boot. Pins the no-cycle property in a fresh interpreter and the re-exported names ten test modules already use. |
| [`test_gateway_kernel_routes.py`](test_gateway_kernel_routes.py) | The twelve kernel routes, pinned before they are moved out of `Handler._api`. Four had no frozen shape and no route-level test at all; four more had only their 403 "REPL disabled" envelope pinned, so swapping the restart and stop bodies passed pytest, the response-shape gate, and the browser smoke run together. Also records two asymmetries as-is rather than tidying them: ten of the twelve answer 200 for an unknown frame id, and `kernel/install` is deliberately ungated by `notebook_repl`. Marked `stubbed_backend` where a service is replaced, so the fabricated responses never reach the frozen contract. |
| [`test_telemetry_schema.py`](test_telemetry_schema.py) | Telemetry may say counts and enumerations, and this is what stops it saying more. Attacks **values**, not keys — `{"error_type": "ValueError"}` and `{"error_type": "FileNotFoundError: /home/y/unpublished/cohort.csv"}` pass the same key check. The leak shapes are taken from what this repo really produces (an exception class defined inside an agent-authored cell, a skill name from SKILL.md frontmatter, an environment directory name, a provider error sentence), crossed against every declared field. Also asserts the package never imports the credential redactor, which exempts absolute paths on purpose. |
| [`test_egress_surface.py`](test_egress_surface.py) | Every module that can open an outbound connection is declared, with a reason. Written as a prerequisite for telemetry but not about it: before this, nothing recorded which modules can reach the network, and "not a single packet leaves the machine" is a claim about the whole tree that cannot be checked one file at a time. Seven modules today. A new one fails with its file and line, so adding a line to the table is a decision while adding `urlopen` to a module is a Tuesday. Listening sockets are deliberately out of scope — a different risk with a different answer. |
| [`test_artifact_manager.py`](test_artifact_manager.py) | Workspace Artifact capture, versioning, and cell promotion. A lot of it is anti-duplication: a provisional version must coalesce rather than become a second version, and an R capture must not run the Python figure probe. Promotion rejects a symlinked output path and embeds figures only as safe data URLs. |
| [`test_artifact_mutation_service.py`](test_artifact_mutation_service.py) | Interactive edit, rename, upload, and delete from the UI. The event shapes are pinned exactly because the frontend reads them, and any path that tries to escape the workspace fails closed. |
| [`test_artifact_repository.py`](test_artifact_repository.py) | The artifact, version, environment, and lineage repositories underneath the manager. The transactional claim is the one to check: if the lineage write fails, the whole `record_cell` transaction rolls back, the version with it. |
| [`test_artifact_scope.py`](test_artifact_scope.py) | Who owns an Artifact when a child frame produced it. Scope is inherited from the root project, an explicit root that conflicts with the known producer is rejected, and a migration repairs sessions written before the rule existed. |
| [`test_backend_import_contract.py`](test_backend_import_contract.py) | A source scan rather than a behavior test. While implementations move into packages, callers may only import the declared facade surface, and no new dependency on an undeclared legacy internal is allowed to appear. |
| [`test_background_cleanup.py`](test_background_cleanup.py) | One test, one guarantee: shutting a session down interrupts and then kills its background kernels, so no worker outlives the session. |
| [`test_bash_authorization.py`](test_bash_authorization.py) | `host.bash` is kernel-local, so the Host never executes it — it can only authorize it. This module is the whole argument that the token is enough. It is bound to command, cwd, generation, and challenge, it is single-use and expiring, and the worker rejects a tampered one before any subprocess is spawned. |
| [`test_bioprobench_skill.py`](test_bioprobench_skill.py) | The BioProBench Skill's two silent scorers. Ground truth encoded as `0`/`1` or `"true"`/`"false"` has to score exactly like a JSON bool, an unrecognized label has to reach `failed_rate` instead of being scored as a confident miss, and a bare correct PQA answer like `0.3` must not be eaten as its own confidence. |
| [`test_capability_state.py`](test_capability_state.py) | Durable Skill and Specialist enablement. The failure it prevents is a Skill that stays disabled in the prompt but not in `search`, or a loader still holding a repository from a closed `Store`. |
| [`test_catalyst_sar_screening.py`](test_catalyst_sar_screening.py) | The catalyst SAR Skill: its documented pipeline and hard lock, its vendored CONTCAR catalog, and the POSCAR parser's edges — a negative scale factor, a truncated file, a structured name that has to be sanitized before it becomes a filename. |
| [`test_cell_dependencies.py`](test_cell_dependencies.py) | Static dependency analysis for Python and R Cells, deliberately allowed to say it does not know. A dynamic namespace write or a parse failure marks the Cell uncertain instead of guessing, and staleness propagates transitively from there. |
| [`test_cell_execution_service.py`](test_cell_execution_service.py) | The Web Cell service, where the transaction ordering lives. An attempt is allocated before the Cell is prepared and finalized even when the worker throws. An R protocol exception shuts down only the lease that was executing, never its replacement. A protocol-only `host.submit_output` is still audited, just not streamed as a Notebook cell. |
| [`test_cell_watchdog.py`](test_cell_watchdog.py) | Timeout recovery for a Cell that will not stop. The subtle case is the one to read: while a Cell is paused waiting for a permission approval, the timeout budget freezes, or a human thinking for a minute would kill the kernel. |
| [`test_checkpoint_state_snapshots.py`](test_checkpoint_state_snapshots.py) | Plan, review, and memory state bound to an immutable checkpoint. Two behaviors are load-bearing: a legacy checkpoint is treated as partial rather than as an empty state that would wipe live data, and corrupt state fails closed instead of restoring half of itself. |
| [`test_cli_contract.py`](test_cli_contract.py) | The CLI surface people script against: entry points, subcommands, options, and help text. `status` reports the local data directory without claiming the daemon is healthy. |
| [`test_compute_nvidia.py`](test_compute_nvidia.py) | The NVIDIA BYOC provider with every `docker` call intercepted by a fake subprocess layer, so no Docker, GPU, or network is needed. | [`test_config.py`](test_config.py) | Layered configuration, and mostly one bug class. A placeholder API key copied out of a template must never be treated as a real key, whether it arrives from the environment or is passed explicitly, and it must not shadow a real per-provider key. |
| [`test_connector_repository.py`](test_connector_repository.py) | MCP connector rows: JSON normalization, ordering, enable and disable, and the `Store` facade that feeds them to the Host MCP service. |
| [`test_context_policy_web.py`](test_context_policy_web.py) | Two tests. A large context output becomes a deduplicated Artifact version instead of a second copy, and the compaction payload is linked back into session history instead of being dropped. |
| [`test_data_background_tools.py`](test_data_background_tools.py) | The data and background tool classes. | [`test_delegation_persistence.py`](test_delegation_persistence.py) | Delegation across a daemon restart. A child that was live when the process died has to be stopped and its lease fenced, not silently adopted by the new generation. Steering, budgets, cascade cancellation, and cleanup on deletion are here too. |
| [`test_delegation_policy.py`](test_delegation_policy.py) | Four tests, one claim: a child's capability and permission policy is enforced, not decorative. An invalid policy fails before it consumes budget, and a nested child can neither widen what its parent allowed nor relax a deny. |
| [`test_delegation_runtime.py`](test_delegation_runtime.py) | Running sub-agents. Budget is shared across the whole tree and reserved atomically between concurrent runners, depth four is an unconditional leaf, and a model reply that arrives after a stop can neither execute nor submit. |
| [`test_dynamic_tool_scopes.py`](test_dynamic_tool_scopes.py) | Dynamic Tools resolved at session, project, and global scope, in that priority order. | [`test_dynamic_tools.py`](test_dynamic_tools.py) | Agent-authored tools are the most direct way to get new code onto the machine, so this is a containment module. | [`test_e2e.py`](test_e2e.py) | Two tests, but the widest stack in the suite: a real Skill loader, a real kernel subprocess, and a real `HostDispatcher`, with only the LLM scripted. The second one checks what the agent actually observes when the kernel raises. |

## Choosing the right place

Put a focused regression assertion here. Put reusable scripted scenarios, fake providers, normalized trajectories, scored evaluations, and reviewed goldens in the [`../harness/`](../harness/) layer. Default tests in both layers stay offline unless their entry point is explicitly marked and invoked as an opt-in smoke.
