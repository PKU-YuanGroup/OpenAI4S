# 生态适配器

[English](README.md)

这里是扩展边界，放的是可选集成：把外部生态接到 OpenAI4S 已有的运行时契约上。导入本包不会给标准库核心引入任何第三方依赖。

## 在架构中的位置

适配器待在系统的边界上，两层循环都不归它管。它可以去驱动已有的外层循环接口或内核接口，但不能自己再实现一套编排，不能绕开 Host 策略，也不能悄悄放宽完成语义的判定。第三方 import 应该放在真正启动适配器的那个位置。

## 文件

| 文件 | 职责 |
| --- | --- |
| [`__init__.py`](./__init__.py) | 标记可选适配器的命名空间。它有意不预先导出任何集成。 |

## 子目录

| 目录 | 职责 |
| --- | --- |
| [`jupyter/`](./jupyter/) | 可选的 Jupyter wire bridge：它驱动已有的 Python/R worker manager。另有纯标准库的 KernelSpec 生成，负责写出用来启动 bridge 的 `kernel.json`。 |

## 扩展契约

- 复用核心已有的 port 或 manager，不要另起一套执行引擎。
- 第三方 import 一律延迟加载，让普通的 `openai4s` 导入仍然只依赖标准库。
- 不支持的集成语义要明确写出来：适配器不会自动等同于 Web workbench。
